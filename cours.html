1 Variables, Types & Mémoire

Une variable est un espace nommé en mémoire (RAM) qui stocke une valeur modifiable. Une constante (const) est liée à une adresse mémoire dont la valeur ne peut pas être réassignée après initialisation.

En JavaScript, il existe 3 mots-clés de déclaration :

  • ▸
    var : ancien, portée globale/fonction, à éviter.
  • ▸
    let : moderne, portée de bloc ({}), valeur modifiable.
  • ▸
    const : portée de bloc, valeur immuable (ne peut pas être réassignée).
let score = 0; // Stack : modifiable
const PI = 3.14159; // Stack : immuable
score = 10; // ✅ OK
// PI = 3; ❌ TypeError : Assignment to constant variable.

1.1 — Coercition de Type (Type Coercion)

JavaScript est un langage à typage dynamique. Il peut convertir automatiquement les types — c'est la coercition. L'opérateur + avec une chaîne de caractères fait de la concaténation, pas une addition.

5 + '5' // → '55' (number converti en string)
5 - '3' // → 2 (string converti en number)
typeof 42 // → 'number'
typeof 'hello'// → 'string'
typeof true // → 'boolean'

1.2 — Égalité Stricte vs Abstraite (=== vs ==)

C'est l'un des pièges les plus classiques en JavaScript. La règle est simple : utilisez toujours ===.

// == (égalité abstraite) : compare SEULEMENT la valeur, ignore le type
5 == '5' // → true ⚠️ DANGEREUX

// === (égalité stricte) : compare la valeur ET le type
5 === '5' // → false ✅ CORRECT
5 === 5 // → true ✅ CORRECT

2 Structures Conditionnelles (if / else if / else)

Une condition évalue une expression booléenne (vrai ou faux) pour décider quelle branche du code exécuter. La logique est : "Si... alors... sinon si... sinon..."

let note = 14;

if (note >= 16) {
  console.log("Très Bien");
} else if (note >= 12) {
  console.log("Assez Bien"); // ← cette branche s'exécute
} else {
  console.log("Insuffisant");
}

⚠️ Edge-Case : Valeurs "Falsy" en JavaScript

Ces valeurs sont automatiquement évaluées comme false dans un if : 0, "" (chaîne vide), null, undefined, NaN, et false lui-même. Tout le reste est "truthy".

if (0) { ... } // ❌ Ne s'exécute PAS
if ("") { ... } // ❌ Ne s'exécute PAS
if (1) { ... } // ✅ S'exécute

3 Structures Itératives (Boucles)

Les boucles automatisent les tâches répétitives. Il est crucial de gérer les conditions d'arrêt pour éviter les boucles infinies qui saturent le CPU.

  • ▸
    for : Idéale quand le nombre d'itérations est connu à l'avance.
  • ▸
    while : S'exécute tant que la condition est vraie. La condition est évaluée avant chaque itération.
  • ▸
    do...while : S'exécute toujours au moins une fois, la condition est évaluée après l'exécution du bloc.
// do...while : le bloc s'exécute UNE FOIS même si la condition est fausse
let i = 100;
do {
  console.log("Exécuté !"); // ← s'affiche malgré i > 5
} while (i < 5);

3.1 — Contrôle de Boucle : break & continue

break termine immédiatement la boucle. continue saute l'itération actuelle et passe à la suivante.

for (let i = 0; i < 10; i++) {
  if (i === 3) continue; // saute i=3
  if (i === 7) break; // arrête la boucle à i=7
  console.log(i); // affiche 0,1,2,4,5,6
}

4 Fonctions

Une fonction est un bloc de code nommé et réutilisable qui accomplit une tâche précise. C'est le principe fondamental de la modularité : au lieu de répéter du code, on le met dans une fonction qu'on appelle autant de fois que nécessaire.

Trois syntaxes existent en JavaScript :

// 1. Déclaration classique (function declaration)
function additionner(a, b) {
  return a + b;
}

// 2. Expression de fonction (function expression)
const multiplier = function(a, b) {
  return a * b;
};

// 3. Fonction fléchée (arrow function) — syntaxe moderne
const diviser = (a, b) => a / b;

console.log(additionner(3, 4)); // → 7

5 Tableaux

Un tableau est une structure de données qui stocke une collection ordonnée d'éléments dans des cases mémoire consécutives. En JavaScript, les index commencent à 0 (zero-indexed).

const tab = ["Alice", "Bob", "Charlie"];

tab[0] // → "Alice" (premier élément, index 0)
tab[1] // → "Bob"
tab[2] // → "Charlie" (dernier élément, index 2)
tab.length // → 3 (nombre d'éléments)

// Parcourir un tableau avec une boucle for
for (let i = 0; i < tab.length; i++) {
  console.log(tab[i]);
}

🔴 Edge-Case Classique

tab[3] dans l'exemple ci-dessus retourne undefined, pas une erreur. Accéder à un index hors limites est silencieux en JS — c'est un bug difficile à détecter.

6 Loi de Pareto (80/20) Appliquée au Code
